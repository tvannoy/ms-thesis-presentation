Steps to Create a Quartus VHDL project

 

Simulink Steps

 

1.      Setup all the paths to the required folders and tool chains on your computer.  This is set in the Matlab script run_me_first.m

2.      Run the run_me_first.m script and change Matlab’s working directory to folder containing this script (Matlab should ask if you are not already in this folder).  This will open the Simulink model with all the necessary information setup to run.

3.      Run the simulation (even if you know it is already correct having been tested & verified) since the model needs the initialization called before the VHDL code is generated.  You should see a pop-up message saying that the verification passed.

4.      Click the button that says Generate VHDL  in the model.  This will take a few minutes since it calls the HDL coder, which ends up running the simulation multiple times.  Ignore warnings for now.  A code generation report will pop up and you can inspect a couple items in the report.

a.       In the outline under Contents and under Timing and Area Report, click on High-level Resource Report.  Note the FPGA fabric resources used since this will give  you an idea if the model will fit into the targeted FPGA.

b.      In the outline under Contents, click on IP Core Generation Report.  Note the location of the IP core folder, which in this case is hdl_prj\ipcore\DataPlane_v1_0.  We will copy the DataPlane_v1_0 folder into the Quartus project we will create (steps below).

c.       You should also see the created tcl script SG_DataPlane_avalon_hw.tcl in \DataPlane_v1_0 if everything generated correctly.

 

Quartus Project Steps

 

5.      Create a local working directory (e.g. \Quartus_Project) and copy a known good Quartus project into this folder.  In our case we will use the AudioMini_Passthrough project for the DE10-Nano board since this is a working passthrough example and we will insert our data plane block into the middle of the streaming passthrough example.

6.      Create a subfolder called \ip in the quartus project folder, i.e. Quartus_Project\ip

7.      Copy the DataPlane_v1_0 folder into the \ip folder, i.e. Quartus_Project\ip\ DataPlane_v1_0

8.      Open the file soc_system.ipx found in the Quartus project folder.  We will need to modify where Quartus is looking for the AD1939 Qsys component.  For now we can just add a hardcoded path in addition to the relative path already in the file (which is most likely not pointing correctly).  In my case I added:
<path path="E:/NIH/NIH_Repo2/Quartus_180/Library/Platform_Designer/AD1939/Source_Files/*" />
to the end of the <path …/> statements and before the </library> line.

a.       Save the modified .ipx file

9.      Open Quartus and open the project file DE10Nano_System.qpf found in \Quartus_Project

10.  Within Quartus, open Platform Designer (found under Tools)

a.       Open soc_system.qsys

11.  In the IP Catalog window, under the Project heading, select simple_gain and click the +Add… button.

a.       Click Finish to accept.

b.      Note that there will be some errors due to connections that still need to be made and that we will fix below.

12.  To connect the simple_gain data plane component in Platform Design, do the following:

a.       Click on the clock line and connect it to outclk0 coming out of the block PLL_using_AD1939_MCLK (you might need to expand this block to see the signal).  Make the connect by clicking on the intersection and making the dot connecting these two clock signals black and filled in, signifying a connection.

b.      Click on the reset line and connect it to clk_reset line coming out of the block clk_hps (which you may need to expand).

c.       Click on the avalon_slave line and connect it to the h2f_lw_axis_master signal in the block hps (which you will need to expand to see the line).

d.      In the block FE_Qsys_AD1939_Audio_Mini_v1_0 disconnect the Line_In to Headphone_Out connection by clicking on the filled dot connecting them.

e.       Click on the avalon_streaming_sink line and connect it to the Line_In signal coming out of the block FE_Qsys_AD1939_Audio_Mini_v1_0

f.        Click on the avalon_streaming_source line and connect it to the Headphone_Out signal coming out of the block FE_Qsys_AD1939_Audio_Mini_v1_0

g.      To get rid of the memory overlap error, click System->Assign Base Addresses

13.  In Platform Design, click the Generate HDL… button

a.       Under the Synthesis section, select VHDL as the language for the design files.

b.      Accept the default output directory path

c.       Click Generate and then close and the system will start generating, which will take a few minutes.

d.      Click close

e.       Exit Platform Designer

14.  In Quartus,

a.       Add a library path the AD1939 VHDL files by

                                                              i.      selecting in Quartus Tools->options

                                                            ii.      click on Libraries under Category on left side

                                                          iii.      In the Project libraries panel, click the … button to browse to the AD1939 folder and click add.  In my case this is at:   E:/NIH/NIH_Repo2/Quartus_180/Library/Platform_Designer/AD1939/Source_Files

                                                          iv.      Click OK

b.      compile the design

 

 

 

 

Debugging Tips

 

Note: when running python scripts from Matlab, and if there are python errors, Matlab doesn’t always propagate and show the python error.  For example,  Matlab just says:

 

Undefined variable "py" or class "py.autogen_tcl.main".

 

However, to show the actual error you need to call python directly:

commandStr = ['python ' mp.codegen_path '\autogen_tcl.py ' infile ' ' outfile ' ' IPcoreVHDLfilesDir];

[status, commandOut] = system(commandStr)
